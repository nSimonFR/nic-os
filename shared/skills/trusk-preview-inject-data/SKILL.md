# Inject test data into a Trusk preview env

Data is generated by the **`trusk-automation`** container, driven by Argo WorkflowTemplates in the **`awf-qa`** namespace on the staging cluster (`trusk-staging-ts`). Any env is targeted by its **`STAGING_NAME`** (`pr-<name>`, `staging`, or `preprod`). Repo: `trusk-argo-workflows` (`docs/workflows/{data-bo,gen-data}.md`).

Prereqs: the preview env is deployed and healthy (see the `trusk-preview-deploy` skill), the **BO pod is Running** (data-bo drives the BO UI via Playwright), and you have kubectl access to `trusk-staging-ts` (cluster-admin).

## Pipeline (order matters)

1. *(if downscaled)* wake the namespace — `workflow-template-wake-up-namespace`.
2. *(optional)* check ArgoCD apps `Synced/Healthy` — `workflow-template-check-argocd-app`.
3. **`data-bo`** — inits BO base data (organisations, contracts) via Playwright against `https://<STAGING_NAME>-bo.trusk.com`. **Run first** — orders need a contract/org to attach to.
4. **`gen-data`** — bulk-generates the entity you ask for.
5. *(optional)* `smoketest-backoffice-template` — validates the env responds.

## Submit (no `argo` CLI — reference the template from a Workflow via kubectl)

```bash
CTX=trusk-staging-ts
# 3) data-bo. Two variants:
#   - do-1361-workflow-template-data-bo    → gated: first step check-argocd-app
#         retries 20×60s until the env's ArgoCD app is Healthy+Synced, then FAILS.
#   - poc-wf-data-bo-without-check-argocd  → SKIPS that gate. Use this on a preview
#         whose aggregate app is Degraded/OutOfSync from bystander apps (very common:
#         service-ratings-typeform init errors, a selfHeal-off backoffice, OutOfSync
#         infra) but is functionally up (BO pod Running). Otherwise data-bo never seeds.
cat <<EOF | kubectl --context $CTX -n awf-qa create -f -
apiVersion: argoproj.io/v1alpha1
kind: Workflow
metadata: { generateName: data-bo-<name>- }
spec:
  workflowTemplateRef: { name: poc-wf-data-bo-without-check-argocd }
  arguments:
    parameters:
      - { name: STAGING_NAME, value: pr-<name> }
EOF

# 4) gen-data (wait for data-bo Succeeded first)
cat <<EOF | kubectl --context $CTX -n awf-qa create -f -
apiVersion: argoproj.io/v1alpha1
kind: Workflow
metadata: { generateName: gen-data-<name>- }
spec:
  workflowTemplateRef: { name: workflow-gen-data }
  arguments:
    parameters:
      - { name: STAGING_NAME,   value: pr-<name> }
      - { name: FUNCTIONALITY,  value: "Commande EDI + rendez-vous + mission" }
      - { name: RUN_COUNT,      value: "3" }
      - { name: POINTS_COUNT,   value: "1" }
EOF
```

Verify template names/params first — they drift (the gated data-bo is currently `do-1361-workflow-template-data-bo`, the no-check one `poc-wf-data-bo-without-check-argocd`):
```bash
kubectl --context $CTX -n awf-qa get workflowtemplates
kubectl --context $CTX -n awf-qa get workflowtemplate workflow-gen-data \
  -o jsonpath='{range .spec.arguments.parameters[*]}{.name}={.value}{"\n"}{end}'
```

## `gen-data` FUNCTIONALITY values → `npm` script

| FUNCTIONALITY | generates |
|---|---|
| `Sites d'expédition` | `gen:sites` |
| `Contrats client` | `gen:contracts` |
| `Organisations` | `gen:organisations` |
| `Création commande` | `gen:edi` (orders via EDI) |
| `Tournées et disponibilités` | `gen:roundtrips` |
| `Factures` | `gen:invoices` |
| **`Commande EDI + rendez-vous + mission`** | order + appointment + **new-mission** (migrated) |
| **`Commande EDI + rendez-vous + mission (legacy)`** | order + appointment + **legacy COURSE** |
| `Tous les scripts` | `gen:all` |

The last two create a full order→appointment→mission chain — exactly what you need to exercise cancel / postpone / replanification. Pick migrated vs legacy for the path under test.

## Monitor

```bash
kubectl --context $CTX -n awf-qa get wf | grep <name>
kubectl --context $CTX -n awf-qa get wf <wf> -o jsonpath='{.status.phase}'   # Running|Succeeded|Failed|Error
kubectl --context $CTX -n awf-qa logs <wf> --container main   # or: follow the pods
```

## Gotchas

- **From-zero preview: bootstrap with `gen-data` first, `data-bo` is NOT self-bootstrapping.** `data-bo` (`data-bo-generate.test.ts`) *looks up* base entities (`getShipmentSiteByCode` → "⚠️ Shipment site not found") and cascade-fails on a virgin preview (~21/33 failed), then hangs on jest open handles so the workflow sits `Running` forever. Auth is fine — it's a missing-baseline problem. Instead run `gen-data` **`Tous les scripts`** (its `gen:*` scripts *POST* sites→contracts→orgs→orders→roundtrips via API and create what's missing), or the granular order `Sites d'expédition` → `Contrats client` → `Organisations` → `Commande EDI + rendez-vous + mission`. Only use `data-bo` once a baseline exists.
- **data-bo before gen-data** — otherwise no contract/org ⇒ order creation fails (only relevant once data-bo works; see the from-zero note above).
- **The gated data-bo stalls on `check-argocd-app`** if the preview's ArgoCD app isn't `Healthy/Synced` — it retries 20×60s then fails, seeding nothing. Symptom: the workflow sits `Running` for ages and the `check-argocd-app` pod logs `❌ Application ArgoCD n'est pas Healthy … Retry dans 60 secondes` (`Tentative N/20`). Previews are routinely Degraded/OutOfSync from bystander apps, so prefer `poc-wf-data-bo-without-check-argocd`. Check the gate's live status: `kubectl -n awf-qa logs -l workflows.argoproj.io/workflow=<wf> -c main --tail=5`.
- data-bo needs the **BO reachable + QA creds** (`trusk-automation-env` configMap+secret in `awf-qa`). If the preview BO crashlooped on env validation, fix it first (it happens — the BO needs `NEXT_PUBLIC_BO_URL` / `NEXT_PUBLIC_TRACK_PAGE_URL` / `NEXT_PUBLIC_TRUSK_BACKOFFICE_URL` injected in a preview namespace).
- Order creation flows **SFTP → centiro EDI ingestion** — the preview's `centiro-edi-warehouse` / `centiro-data-grabber` must be running.
- Volume = `RUN_COUNT × POINTS_COUNT`; too large ⇒ pod timeout (`activeDeadlineSeconds`). Start small (3×1).
- Outputs (CSV/logs) land on SFTP under `data/staging_k8s/<STAGING_NAME>` for PR envs.
- There's a `psql_truncate_preview` configmap to wipe a preview DB between runs if you need a clean slate.
